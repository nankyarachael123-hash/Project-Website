🔥 Firebase Setup — 3 minutes, free forever
  1. Go to console.firebase.google.com → Add project → name it (e.g. "qc-dashboard") → Continue
  2. In your project: click Firestore Database → Create database → choose Start in test mode → Next → Enable
  3. Click the ⚙ gear icon → Project settings → scroll to Your apps → click </> Web → register app → copy the firebaseConfig object
  4. Paste it in the box below and click Save & Connect

Using this on GitHub Pages

Data is stored in your browser's localStorage — it persists when you close and reopen the file on the same device, but won't sync across devices automatically. GitHub Pages is read-only and cannot write back to your repo.

Best practice: after each session, click ⬇ Export JSON and commit the file to your repo (e.g. data/qc_data.json). On any new device, click ⬆ Import JSON to restore. This gives you full version control of your data alongside your code.
